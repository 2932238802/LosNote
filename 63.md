4. 详细 介绍一下

std::to_chars_result to_chars(char* first, char* last, T value, int base = 10);
- 第一个参数 是 提供缓冲区的 起始位置
- 第二个参数 是 指向你提供的缓冲区的结束位置 
- 第三个参数 是你想要转换的数字
- 第四个参数 是 转换的基数 可以是 2 ~ 36的任意值


- 设计哲学：
  - 它不自己申请内存，你必须提供一个预先分配好的字符缓冲区，它只负责往里面写
  - 通过 返回值 to_chars_result 来告诉你 结果 是不是 空间不足
  - 生成的数字 是 固定的 不会 收到操作系统语言的影响
  - 通过 返回一个 错误码 来报告失败 而不是抛出异常 这使得它 在不能使用异常的环境中也能工作



- to_chars_result 的结构形式

```c++
struct to_chars_result {
    char* ptr;      // 转换后，指向写入的最后一个字符的下一个位置
    std::errc ec;   // 错误码
};
```


```C++
#include <charconv>

#include <iostream>
#include <string_view>
#include <system_error>
using namespace std;

int main() {
  int number = 100;

  char arr[20]{};

  auto res = std::to_chars(arr, arr + sizeof arr, number);

  if (res.ec == std::errc()) {
    // 如果是 成功 读取内部的值 就可以
    size_t lenth = res.ptr - arr;

    std::string_view vi(arr, lenth);

    std::cout << vi << std::endl;
  }

  return 0;
}
```





5. 如何理解 std::errc()？
- std::errc 是一错误清单 没有错误 就是 用 这个 的默认构造 std::err();
- 默认构造 内部就是 0 。 按照约定 值为0的错误码就是 表示成功
