1. 运行rust

rustc main.rs



2. cargo run 

cargo run 命令会以 Cargo.toml 文件所在的目录作为项目的根目录，然后遵循 Cargo 的标准项目布局约定，找到 src/main.rs 文件，将其编译成一个可执行文件（如果代码有变动的话），最后运行这个可执行文件。程序的入口点就是 src/main.rs 文件里的 main 函数



3. cargo check

快速检查代码 确保可以编译 但不产生可执行文件



4. cargo run 的一些情况

如果src名字被改变
如果main.rc的名字被改变
如果main函数不在main.rc

会怎么样 针对这个指令？



- 第一种情况 会报错 因为这个是约定

  - 可以通过修改 Cargo.toml

  - ```rust
    [package]
    name = "my_project"
    version = "0.1.0"
    edition = "2021"
    
    # 添加下面的 [[bin]] 部分
    [[bin]]
    name = "my_project" # 这是你希望生成的可执行文件的名字
    path = "source/main.rs" # 明确指定入口文件的路径
    
    ```
  
- 第二种情况 还是会报错

  - ```rust
    [package]
    name = "my_project"
    version = "0.1.0"
    edition = "2021"
    
    [[bin]]
    name = "my_project"
    path = "src/app.rs" # 把路径指向新的文件名
    
    ```

- 第三种情况

  - 会失败 必须要有main函数
