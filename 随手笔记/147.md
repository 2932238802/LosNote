    pub mod request {
        include!(concat!(env!("OUT_DIR"), "/auth_request.rs"));
    }

    pub mod response {

- rust 关于 protobuf的包含问题

- env!

  - Cargo 在执行 `build.rs` 时，会创建一个专门存放生成文件的临时目录。
  - 这个目录的路径非常长且随机（比如 `target/debug/build/项目名-hash/out`）。
  - `env!("OUT_DIR")` 就会返回这个具体的路径字符串

- ### `concat!`

  - 在编译的时候 把几个字符串串在一起

- include!

  - 在编译的时候 读取指定文件的路径
  - 并把这些内容 原封不动地替换到当前位置
