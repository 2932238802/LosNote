2. 

restful 的核心原则或约束有哪些？

- 客户端和服务器是分离的，客户端负责用户界面，服务器负责数据存储和业务逻辑
- 无状态就是说，服务器不应该保存任何客户端会话的状态信息
- 每一次从客户端发来的请求，都必须包含所有服务器理解和处理该请求所需要的全部信息
- 服务器处理完请求之后就会 清楚 这一次的交互 便于负载均衡
- 统一接口
  - 资源标识 url
  - 通过表述来操作资源，客户端通过获取资源的表示，像json或者xml来操作资源
  - 每个信息都包含足够的信息来描述如何处理

4. 进程和线程的区别？



- 进程是资源分配的基本单位。每个进程都拥有自己独立的系统资源，包括独立的地址空间、文件描述符、全局变量等。一个进程的崩溃通常不会直接影响到其他进程。

- [ ] 11_17



- 线程：是CPU调度的基本单位。线程本身不拥有系统资源，它只拥有一点儿必不可少的、保证独立运行的资源，比如程序计数器、寄存器和栈。它与同属一个进程的其他线程共享该进程的全部资源。
  内存空间：

- [x] 11_17



- 进程：拥有独立的、受保护的内存地址空间。一个进程无法直接访问另一个进程的内存，除非使用进程间通信（IPC）机制。

- [x] 11_17





- 线程：同一进程内的所有线程共享同一个内存地址空间（包括代码段、数据段、堆）。这意味着线程之间可以直接读写全局变量和堆上的数据，但也因此需要同步机制（如互斥锁）来避免数据竞争。
  通信方式：

- 进程：进程间通信（Inter-Process Communication, IPC）相对复杂且开销较大。常用的方式有管道（Pipe）、消息队列（Message Queue）、共享内存（Shared Memory）、信号量（Semaphore）和套接字（Socket）等。

- [x] 11_17





- 线程：线程间通信（Inter-Thread Communication）非常方便快捷。因为它们共享内存，所以可以直接通过读写全局变量或共享数据结构来通信。不过，这种便利性也带来了线程同步的挑战。

5. 

Qt的事件循环工作原理

- Qt的GUI 应用程序启动之后，调用 exec 进入事件循环
- 事件循环是一个持续运行的循环，负责从操作系统的事件队列中获取事件，点击，输入等，并将这些事件放到Qt自己的事件队列中
- 事件循环会按顺序从Qt事件中取出事件，并将它们分发，给相应的QObject
- 等待事件->分发事件->处理事件 不断重复 直到应用程序退出
