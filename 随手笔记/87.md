字面量和字符串的区别

#### A. 字符串字面量 / 字符串切片 (`&str`)

这就是你直接写在代码里的 `"..."`。

- **存储位置**：通常存储在编译后的**二进制程序文件**中（静态内存区）。

- **特性**：**不可变**（你不能修改写在代码里的字面量）、**固定长度**。

- **代码**：

  rust

  ```rust
  let s = "Hello"; // s 的类型是 &str
  ```

#### B. String 类型 (`String`)

这是一个**智能指针**，指向堆内存（Heap）中的数据。

- **存储位置**：**堆内存** (Heap)。

- **特性**：**可变**（可以修改内容）、**可增长**（可以变长或变短）、拥有所有权。

- **代码**：

  rust

  ```rust
  let s = String::from("Hello"); // s 的类型是 String
  ```

```vue
<script setup>
import { ref } from "vue";
import { invoke } from "@tauri-apps/api/core";

const greetMsg = ref("");
const name = ref("");

async function greet() {
  // Learn more about Tauri commands at https://tauri.app/develop/calling-rust/
  greetMsg.value = await invoke("greet", { name: name.value });
}
</script>
```

问题：

​	1. invoke 的第二个参数 是不是取默认值？

​	2. import { invoke } from "@tauri-apps/api/core"; 这个一步是不是就是导入了这个项目里面invoke的所有rust的接口？

- 答：第一个问题：第二个参数是传递给greet的参数，
- 答：第二个问题：invoke是通用的，它是一个Tauri提供的API函数，用来从前端发送消息给后端
  - 它是动态调用的，它只是把字符串"greet"发送给Tauri懂得核心层，核心层再去Rust端查找有没有注册过叫“greet”的命令
- 真正的注册在 Rust 端： Rust 接口必须在 `src-tauri/src/main.rs` 的 `.invoke_handler(tauri::generate_handler![greet])` 这里注册了，`invoke` 才能打通。如果 Rust 没注册，前端调用 `invoke("greet")` 会报错
