3. [rust]

- Package Cargo的特性 让你构建 测试 共享Crate
  - 包含一个 cargo.toml
  - 只能包含 0-1 library crate
  - 可以包含多个 binary crate
  - 但必须至少包含一个 crate
- Crate 一个模块树 它可产生一个library 或 可执行文件
- Module，use 让你控制代码的组织，作用域，私有路径

- cargo 默认会把 src/main.rs 作为binary crate 的crate root
- src/lib.rs 
  - 一个package 包含一个 library crate
  - library crate 的 crate root
  - crate 名与 package 名相同
- 一个 Package 可以包含 src/main.rs 和 src/lib.rs
- 一个 binary crate 一个 library crate
- 一个package 可以有多个binary crate 
  - 文件放在 src/bin 
  - 每个文件 是 单独的binary crate

4. [rust]

```rust
smart_home/
├── Cargo.toml
└── src/
    ├── main.rs                 <-- [Level 0] 二进制入口
    ├── lib.rs                  <-- [Level 0] 库入口 (CEO)
    │
    ├── network/                <-- [Level 1] 网络层
    │   ├── mod.rs              <-- network 模块的管理员
    │   ├── wifi.rs
    │   └── protocol/           <-- [Level 2] 协议层
    │       ├── mod.rs
    │       ├── http.rs
    │       └── mqtt/           <-- [Level 3] MQTT 协议细节
    │           ├── mod.rs
    │           ├── connect.rs
    │           └── publish.rs  <-- [Level 4] 我们要调用的目标！
    │
    └── devices/                <-- [Level 1] 设备层
        ├── mod.rs
        ├── light.rs
        └── sensor/             <-- [Level 2] 传感器
            ├── mod.rs
            ├── temp.rs
            └── humidity.rs
```



你可以把模块入口文件放在文件夹**外面**，名字和文件夹**同名**

```rust
src/
├── lib.rs
├── network.rs    <-- 这个文件代替了上面的 network/mod.rs
└── network/      <-- 这个文件夹只存放子模块文件
    ├── wifi.rs
    └── http.rs
```

这种写法更好
