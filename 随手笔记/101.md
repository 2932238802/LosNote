- ```rust
  let file_result = OpenOptions::new()
          .create(true) // 如果文件不存在，就创建它 (非常重要！)
          .write(true)  // 我要写入
          .append(true) // 关键：开启追加模式
          .open(path);  // 打开文件
  
      match file_result {
          Ok(mut file) => {
              // 2. 写入内容
              // 建议手动加个换行符，不然多次追加会连在一起变成 "HelloWorldAgain"
              let final_content = format!("{}\n", content);
              
              match file.write_all(final_content.as_bytes()) {
                  Ok(_) => println!("✅ 追加写入成功"),
                  Err(e) => println!("❌ 写入数据失败: {}", e),
              }
          },
          Err(e) => println!("❌ 无法打开文件: {}", e),
      }
  ```





OpenOptions::new()

这个返回的本质是什么

- 本质是一个 **`OpenOptions` 结构体**
- 所有 勾选 默认是 否
- 接下来的一系列链式调用，就是在单子上打勾：
  - `.read(true)` -> ☑️ 我要读取
  - `.write(true)` -> ☑️ 我要写入
  - `.create(true)` -> ☑️ 如果没有文件，请帮我新建一个
  - `.append(true)` -> ☑️ 我要接着末尾写，别删旧内容
  - `.truncate(true)` -> ☑️ 把里面内容清空
