1. QML 还是 QObject

| 特性           | Qt Widgets                                                   | QML                                                          |
| :------------- | :----------------------------------------------------------- | :----------------------------------------------------------- |
| **编程范式**   | **命令式** (Imperative)，使用C++代码构建和控制UI             | **声明式** (Declarative)，使用类似JSON的语法描述UI结构和状态 |
| **开发效率**   | 复杂界面构建相对繁琐，但逻辑控制直接                         | 界面原型设计和迭代速度极快，尤其适合UI/UX设计师介入          |
| **性能**       | 经过长期优化，原生控件性能非常高，适合数据密集型、需要极致响应的应用 | 渲染基于场景图(Scene Graph)，对动画和图形效果优化好。但在大量控件的场景下，启动和对象创建开销可能高于Widgets |
| **外观与动画** | 依赖QSS和自定义绘制(`paintEvent`)，实现复杂动画和现代UI效果需要大量编码 | 天生为动态、流畅的UI设计，实现动画、过渡效果非常简单自然     |
| **后端集成**   | 与C++业务逻辑无缝集成，因为它们都在C++层                     | 需要通过清晰的接口（上下文属性、注册类型）与C++后端交互，强制了前后端分离 |
| **目标平台**   | 传统桌面应用的王者                                           | 移动端、嵌入式设备的首选，也适用于需要现代感和触摸体验的桌面应用 |

2. 

为什么 需要线程池

- 对于大量短时任务，如果每个任务都创建一个新线程：
  - 就会 有高昂的创建和销毁
  - 资源耗尽
  - 上下文切换 频繁

- 线程池 通过复用 固定数量的线程来解决问题 提高性能和稳定性
