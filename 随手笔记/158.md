1. [C++ windows]

    SetProcessDpiAwarenessContext(DPI_AWARENESS_CONTEXT_PER_MONITOR_AWARE_V2);

如何理解这宏和这个参数 以及对应的意义？



- | **1. 昏迷 (Unaware)**           | `DPI_UNAWARE`              | 告诉程序 DPI 永远是 96。如果屏幕是 4K，Windows 先让程序画个小的，然后拉伸。 | **模糊**，但坐标永远是 96 对应的。                     |
  | ------------------------------- | -------------------------- | ------------------------------------------------------------ | ------------------------------------------------------ |
  | **2. 系统觉醒 (System Aware)**  | `SYSTEM_DPI_AWARE`         | 程序启动时检测**主显示器**的 DPI。所有屏幕都按这个 DPI 画。  | 在主屏清晰，副屏如果缩放不同就会**模糊**。             |
  | **3. 显示器觉醒 V1 (PM Aware)** | `PER_MONITOR_AWARE`        | 窗口拖到哪个屏幕，就按哪个屏幕的 DPI 画。                    | **清晰**。但标题栏、菜单栏、对话框不会自动缩放。       |
  | **4. 顶级觉醒 V2 (PM V2)**      | **`PER_MONITOR_AWARE_V2`** | 窗口动态感知每个显示器的 DPI。                               | **最清晰**。包括标题栏、公共控件等所有内容都自动适配。 |

如何理解 HDC？HWND？

- hwnd 就是 窗口的句柄
  - 内核模式下 它对应的一个名为 WND的内部结构体 ， 记录了窗口的Z- 序 ， 类信息，进程ID，线程ID，窗口过程函数地址以及窗口相对于屏幕或父窗口的逻辑坐标
- ReleaseDC(window,hdc);
  - 就是 可以断开 window hdc的绑定关系

auto add(std::integral auto a, std::integral auto b) {
    return a + b;

如果这么写
那么 规范是什么？（语法）
这里的 std::integral是一个定语？
    
    
    
int main()
{


    auto add = [](
        std::integral auto a,
        std::integral auto b 
    ){

        return a +b;
    };

    cout << add(1,2) <<endl;   // 3
}
```



- 都是对的

- std::integral 就是一个定语 一个修饰

- Concepts 专门约束 模板参数

- c++之后

  - ```c++
    void print(auto x) { ... }
    ```

  - 本质上就是

    - ```c++
      template<typename T>
      void print(T x) { ... }
      ```
