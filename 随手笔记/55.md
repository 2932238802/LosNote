2. 如何理解 循环引用

```c++
#include <iostream>
#include <memory>

class NodeB; // 前向声明

class NodeA {
public:
    std::shared_ptr<NodeB> b_ptr;

    NodeA() { std::cout << "NodeA constructor" << std::endl; }
    ~NodeA() { std::cout << "NodeA destructor" << std::endl; }
};

class NodeB {
public:
    std::shared_ptr<NodeA> a_ptr;

    NodeB() { std::cout << "NodeB constructor" << std::endl; }
    ~NodeB() { std::cout << "NodeB destructor" << std::endl; }
};

void test_cycle() {
    std::cout << "--- Entering test_cycle ---" << std::endl;

    // 1. 创建两个对象，此时它们的引用计数都为 1
    auto a = std::make_shared<NodeA>(); // a_ptr 指向 NodeA 对象, A 的引用计数 = 1
    auto b = std::make_shared<NodeB>(); // b_ptr 指向 NodeB 对象, B 的引用计数 = 1

    std::cout << "A's use_count: " << a.use_count() << std::endl; // 输出 1
    std::cout << "B's use_count: " << b.use_count() << std::endl; // 输出 1

    // 2. 互相引用，形成循环
    a->b_ptr = b; // NodeA 对象内部的 b_ptr 指向 NodeB, B 的引用计数变为 2
    b->a_ptr = a; // NodeB 对象内部的 a_ptr 指向 NodeA, A 的引用计数变为 2

    std::cout << "A's use_count after cycle: " << a.use_count() << std::endl; // 输出 2
    std::cout << "B's use_count after cycle: " << b.use_count() << std::endl; // 输出 2

    std::cout << "--- Exiting test_cycle ---" << std::endl;
} // test_cycle 函数结束

int main() {
    test_cycle();
    std::cout << "--- After test_cycle, program ends ---" << std::endl;
    return 0;
}
```

3. std::move的作用是什么

- 本身不做任何移动操作，它的作用就是将一个左值无条件强制转换为右值引用。
- 它让新对象的指针指向旧对象所管理的资源
- 然后将就对象的指针置为nullptr

- | 业务操作 | 传统 RPC 风格 (URL 是动作) | RESTful 风格 (URL 是资源) |

  | 更新用户 | `POST /updateUser`         | `PUT /users/123`          |
  | 删除用户 | `POST /deleteUser`         | `DELETE /users/123`       |
