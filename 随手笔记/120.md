4. 

```dockerfile
version: '3.8'
services:
  lostrans-backend:
    build:
      context: ./backend-rust
      dockerfile: Dockerfile
    container_name: lostrans-app
    ports:
      - "3000:3000"
    volumes:
      - ./backend-rust/data:/app/data
    environment:
      - DATABASE_URL=/app/data/lostrans.db
      - RUST_LOG=info
    restart: always

```



如何理解？
逐行介绍一下
以及对应的语法知识
越详细越好





- version 是 Docker Compose 的版本信息

- services  是 入口

- lostrans-backend 是服务的名字

- `build:`

  - 语法：指定如何构建镜像
  - context 构建上下文
  - dockerfile  默认就是 `Dockerfile`。如果你起名叫 `My.Dockerfile`，这里就要对应修改

- container_name  这个是默认的文件名字
- ports

  - 左侧是容器内 端口 右侧是 宿主机的端口

- `volumes:`
  - 语法：`"宿主机路径:容器内路径"`（数据卷挂载）
  - `./backend-rust/data`：你 Windows 硬盘上的文件夹
  - `/app/data`：容器内部 Linux 系统里的文件夹
