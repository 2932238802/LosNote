- 用户模式 -> 内核模式
- sys_socket
- 并在 内核空间的 堆区 为该套接字分配内存 创建一个 struct socket 内核对象



- 执行 socket的时候
  - 内核中形成 如下的逻辑链条
  - 用户的 fd -> struct file -> struct socket -> struct udp_sock(tcp_sock)
    - 第一层 struct file
      - 因为 让这个行为像 文件一样 read write close
    - 第二层 struct socket
      - 它是 上层协议（TCP/UDP） 与 下层文件系统 之间的桥梁
