virtual 关键字 的核心作用是什么？

- 当一个积累的成员 被声明为 virtual的时候 ，通过基类指针或者引用调用这个函数的时候，程序会根据指针或应用实际指向的对象类型来决定调用哪个版本的函数，而不是根据指针或引用本身



- [ ] 11_17

- 实现原理
  - 虚函数表
    - 当一个类 中包含 虚函数的时候 ，编译器会为该类 生成一个静态的虚函数表。 函数指针数组，存放着类中所有虚函数的地址
    - 还有一个虚函数指针，在对象创建的时候被初始化，指向所属类的虚函数表
    - 会有一个动态绑定的过程，在运行的时候确定调用哪个函数版本

TCP 协议简历连接的三次握手？

第一次握手: 客户端向服务器发送一个SYN同步序列编号报文段，其中包含一个初始序列号seq=x,并进入 SYN_SENT 状态

- [ ] 11_17





第二次握手: SYN + ACK 服务器收到SYN报文段之后，如果统一建立连接，会回复一个 SYN + ACK 报文段。 其中包含确认号 ack = x + 1, 以及服务器自己的初始序列号 seq = y 。 服务器进入 SYN_RCVD 状态

- [x] 11_17



第三次握手: 客户端收到服务器的 SYN+ACK报文段后，会向服务器发送一个ACK报文段，其中包含确认号为 ack = y+1 . 发送完毕之后，客户端和服务器都进入 established 状态，连接建立成功
