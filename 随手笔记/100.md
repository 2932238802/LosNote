2. 

```rust
#[derive(Subcommand)]
这个宏的 基本上都是 在 枚举上面么？
```



- **`#[derive(Parser)]`** -> 用在**结构体**上 (整个程序的入口)。
- **`#[derive(Args)]`** -> 用在**结构体**上 (一组参数的集合)。
- **`#[derive(Subcommand)]`** -> 用在**枚举**上 (子命令的选择)

- [ ] 1_18

4. [rust]

```rust
let args_for_clap = std::iter::once("todo").chain(parts.into_iter());
如何理解？
```

- rust 的 链式调用

- parts.into_iter()  切片数组的 into_iter() 把切片数组转换成一个迭代器

- `std::iter::once("todo")`  创建一个只能吐出一个元素 的 迭代器

```rust

       let mut input_content = String::from("");

        // 解析一下
        let parts :Vec<&str> = input_content.trim().split_whitespace().collect();

```

- split_whitespace 是按照空格进行分割 返回迭代器
